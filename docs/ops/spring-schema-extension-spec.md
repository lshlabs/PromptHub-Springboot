# Spring 아키텍처 마이그레이션에 따른 스키마 확장 명세서

## 목적

이 문서는 Django 레거시 구조에서 벗어나 **Spring Boot 환경에 맞춘 보안성과 확장성 강화**를 위해
의도적으로 도입된 스키마 확장 항목을 공식적으로 정의하고, 동등성 판단 대상에서 제외한다.

## 승인 사항 (동등성 판단 제외)

아래 두 항목은 **의도적 스키마 불일치(B등급)**로 확정하며, 롤백하지 않는다.

### 1) `users.google_sub` (SSO 고유 식별자)

- 목적: Google SSO 계정의 불변 식별자(`sub`)를 저장해 사용자 매칭 안정성 강화
- Django 기준: 해당 필드 없음
- Spring 기준: 선택적 확장 필드(UNIQUE), 로그인/연동 안정성 향상
- 결정: **동등성 판단 대상에서 제외, 유지**

**리스크/점검 포인트**
- 중복 `google_sub` 삽입 방지(UNIQUE 유지)
- 로컬 계정과 SSO 계정 병합 정책 문서화
- 탈퇴/계정 연결 해제 시 정책 명확화

### 2) `auth_tokens.token_type` (토큰 유형 구분)

- 목적: Access/Refresh 등 토큰 유형을 명시적으로 구분해 보안성과 제어성 강화
- Django 기준: DRF Token은 단일 타입
- Spring 기준: enum 기반 타입 분기 + 정책 확장 기반
- 결정: **동등성 판단 대상에서 제외, 유지**

**리스크/점검 포인트**
- 토큰 발급/회수 로직에서 `token_type` 누락 방지
- 재발급/폐기 경로에서 타입별 분기 테스트 보강
- 만료 정책(`expires_at`)과의 일관성 유지

## 운영 규칙

- 위 확장 항목은 **“Spring 아키텍처 마이그레이션에 따른 스키마 확장”**으로 공식 고정한다.
- Django와의 1:1 동등성 판정에서 **예외 항목**으로 처리한다.
- 스키마/인증 정책 변경 시 반드시 본 문서에 변경 이력을 남긴다.
